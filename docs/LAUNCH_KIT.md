## One-Line Pitch

## Short Post

I built CommandNest, an open-source desktop AI assistant that opens with a global shortcut and streams responses through OpenRouter. It started as a native macOS menu bar app and now includes Windows/Linux builds.

It can also act locally when you ask it to work with files, folders, shell commands, or apps. Write/move/trash/shell/open actions show confirmation prompts by default.

Repo and downloads: https://github.com/vininhosts/CommandNest

## Longer Post

CommandNest is a lightweight desktop AI assistant for people who want something closer to a command palette than a full chat app.

- macOS native Swift menu bar app
- Windows/Linux Electron edition
- Global shortcut: Option+Space on macOS, Alt+Space on Windows/Linux
- OpenRouter Chat Completions with streaming
- Free router default model
- Searchable model picker
- Markdown rendering and separate Thinking panel
- Local agent tools for reading/writing files, organizing folders, opening apps/URLs, and shell commands
- API keys stored in Keychain or OS secure storage

The goal is a small open-source assistant you can run as a normal desktop app and inspect end to end.

GitHub: https://github.com/vininhosts/CommandNest

## Safety Notes

- The app is powerful because it can act on local files and shell commands.
- Destructive or mutating local actions should remain visible and confirmable.
- Public builds are unsigned/ad-hoc unless signed with platform certificates.
